<div class="markdown prose w-full break-words dark:prose-invert dark" bis_skin_checked="1"><ol><li><p><strong>Algoritmii RSA, ElGamal și Diffie-Hellman</strong>:</p><ul><li><strong>RSA</strong>: Este un algoritm de criptografie asimetrică utilizat pentru securizarea transferului de date. Se bazează pe principiul că este ușor să multiplicați două numere mari, dar este greu să descompuneți produsul în factorii săi primari originali.</li><li><strong>ElGamal</strong>: Este un alt algoritm de criptografie asimetrică. Se bazează pe logaritmul discret și este folosit mai ales pentru criptarea datelor, mai puțin pentru semnături digitale.</li><li><strong>Diffie-Hellman</strong>: Este o metodă de a genera în siguranță o cheie secretă comună peste un canal de comunicare nesecurizat. Nu este un algoritm de criptare în sine, ci mai degrabă un protocol de schimb de chei.</li></ul></li><li><p><strong>Pașii de generare a cheilor după RSA</strong>:</p><ol><li>Alegeți două numere prime mari, denumite p și q.</li><li>Calculați n = p * q.</li><li>Calculați φ(n) = (p-1)(q-1).</li><li>Alegeți un număr întreg e astfel încât 1 &lt; e &lt; φ(n) și e să fie coprim cu φ(n).</li><li>Calculați d, inversul modular al lui e în raport cu φ(n).</li><li>Perechea (e, n) este cheia publică, iar (d, n) este cheia privată.</li></ol></li><li><p><strong>Cum utilizăm cheile RSA pentru a cripta și decripta un mesaj</strong>:</p><ul><li>Criptare: Pentru a cripta un mesaj, transformați mesajul într-un număr m într-un mod standard. Apoi calculați mesajul criptat c = m^e mod n.</li><li>Decriptare: Pentru a decripta, folosiți cheia privată pentru a calcula m = c^d mod n.</li></ul></li><li><p><strong>Pașii după algoritmul Diffie-Hellman pentru a ajunge la o cheie comună între două părți</strong>:</p><ol><li>Alegeți un număr prim mare p și o bază g.</li><li>Prima parte alege un număr secret a și trimite A = g^a mod p celeilalte părți.</li><li>A doua parte alege un număr secret b și trimite B = g^b mod p primei părți.</li><li>Prima parte calculează cheia secretă K = B^a mod p.</li><li>A doua parte calculează cheia secretă K = A^b mod p.</li><li>Ambele părți vor avea acum aceeași cheie secretă K.</li></ol></li></ol><hr><br>
<ul><li><p><strong>Criptografia Simetrică</strong>: Folosește aceeași cheie pentru criptarea și decriptarea datelor. Aceasta înseamnă că atât expeditorul, cât și destinatarul trebuie să aibă această cheie, ceea ce poate ridica probleme de securitate în distribuirea cheii.</p></li><li><p><strong>Criptografia Asimetrică</strong>: Utilizează un set de două chei - una publică și una privată. Cheia publică este distribuită liber pentru criptare, în timp ce cheia privată este păstrată secretă și folosită pentru decriptare. Acest sistem asigură o securitate sporită, deoarece chiar și dacă cheia publică este cunoscută, fără cheia privată, datele criptate rămân inaccesibile.</p></li></ul></div>
